## Implementation Summary

### mesh serve
- Loads YAML config from --config flag or default path
- Creates daemon via daemon.New(cfg)
- Starts daemon blocking via d.Start(ctx)
- Handles "already running" error gracefully

### mesh stop
- Loads config to get pid_file path
- Reads PID file, validates PID
- Sends SIGTERM to process
- Waits for process exit with --timeout flag (default 30s)
- Sends SIGKILL if timeout exceeded
- Handles errors: no PID file, invalid PID, process not found

### mesh status
- Loads config to get pid_file path
- Reads PID file
- Checks if process is alive via Signal(0)
- Displays "running (pid N)" or "stopped"
- Note: health endpoint query not implemented because daemon binds to random port (127.0.0.1:0)

## Tests Added
- TestServeCommandStartsDaemon: verifies daemon starts, writes PID file, cleans up on stop
- TestStopCommandKillsDaemon: verifies stop sends SIGTERM/SIGKILL, process exits
- TestStopCommandNotRunning: verifies error when no PID file exists
- TestStatusCommandRunning: verifies status shows running with PID
- TestStatusCommandStopped: verifies status shows stopped when no PID file
