# Task 9: Implement execute_command MCP tool — Evidence

## Changes Made

### Files Modified
- `internal/mcp/handlers.go` — Replaced stub with real implementation
- `internal/mcp/mcp_test.go` — Added comprehensive tests
- `internal/body/manager.go` — Added Exec method to BodyManager

### Implementation Details

1. **execute_command handler** (`internal/mcp/handlers.go`):
   - Parses `body_id`, `command` (array), and optional `timeout_seconds`
   - Validates body exists in store
   - Checks body state is Running before executing
   - Supports configurable timeout (default 30s)
   - Uses `context.WithTimeout` for deadline enforcement
   - Calls `bodyMgr.Exec()` which delegates to substrate adapter
   - Returns `{stdout, stderr, exit_code}` on success
   - Returns RPC errors for: body not found, not running, timeout, exec failure

2. **BodyManager.Exec** (`internal/body/manager.go`):
   - Added `Exec(ctx, bodyID, cmd)` method
   - Validates body state is Running
   - Delegates to `adapter.Exec()` with the body's instance handle

3. **Tests** (`internal/mcp/mcp_test.go`):
   - `TestExecCommandSuccess` — exec on running container returns stdout/exit_code
   - `TestExecCommandNotRunning` — exec on stopped body returns "not running" error
   - `TestExecCommandTimeout` — exec with 1s timeout on sleep 10 returns timeout error
   - `TestExecCommandEmptyCommand` — empty command array returns validation error
   - `TestExecCommandBodyNotFound` — nonexistent body returns "not found" error
   - `TestExecCommandNoBodyManager` — no body manager returns "not available" error
   - Updated mock adapter to support timeout simulation (sleep command respects context)
