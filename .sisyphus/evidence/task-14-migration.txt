IMPLEMENTATION SUMMARY
----------------------

Step 2 (provision) - stepProvision():
- Idempotency: checks if mig.newHandle already set, or if migration record
  CurrentStep >= 2 and recovers handle from body record's InstanceID
- Inspects source container to get metadata (image, workdir, env, cmd)
- Creates target container via adapter.Create() with same spec
- Stores target handle in mig.newHandle
- Rollback on persist failure: destroys target container, clears handle
- Error handling: wrapped with fmt.Errorf for context

Step 3 (transfer local) - stepTransfer():
- Idempotency: checks if migration record CurrentStep >= 3
- Validates prerequisites: snapshotPath and newHandle must be set
- Opens snapshot tarball from disk
- Imports into target container via adapter.ImportFilesystem()
- Uses Overwrite: true for import opts

Step 1 (export) - stepExport():
- Also enhanced to actually write exported filesystem data to disk
- Previously only created store record without writing file
- Now writes to /tmp/mesh-snapshot-{uuid}.tar.zst

State Machine Changes:
- Added Running -> Running transition (for idempotent resume)
- Added Error -> Migrating transition (for retry after failure)
