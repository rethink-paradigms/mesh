# Discovery Folder Structure

> Map of all files in the discovery/ folder and related governance files.
> Updated when discovery/ structure changes.

## Top-Level Files

| File | Purpose |
|------|---------|
| `INDEX.md` | Project dashboard — decisions made, open questions, research status |
| `intent.md` | What we're building and why. Stable. Rarely changes. |
| `constraints.md` | Hard boundaries (C1-C6). Non-negotiable. |

## Directories

### `state/` — Machine-Readable State

| File | Source | Purpose |
|------|--------|---------|
| `decisions.md` | **Generated from DB** | Product decisions (D1+). Don't hand-edit. Regenerate with `generate.py decisions-md`. |
| `governance-decisions.md` | **Generated from DB** | Governance decisions (D-GOV1+). Don't hand-edit. Regenerate with `generate.py governance-md`. |
| `open-questions.md` | **Generated from DB** | Open questions (Q1+). Regenerate with `generate.py questions-md`. |
| `personas.md` | Hand-written | Agent personas (A1-A5). Validated against designs. |

|----------|---------|

## Governance Infrastructure

| File | Purpose |
|------|---------|
| `.mesh/governance.db` | SQLite graph DB — entities, edges, sessions. Source of truth for decisions/questions/constraints. |
| `~/.agents/skills/mesh-nav/scripts/gov.py` | Entity/edge CRUD and graph traversal |
| `~/.agents/skills/mesh-nav/scripts/session.py` | Session logging and retrieval |
| `~/.agents/skills/mesh-nav/scripts/generate.py` | Markdown generation from DB |

## Generated Files (Don't Hand-Edit)

These files are regenerated from `.mesh/governance.db`:

1. `discovery/state/decisions.md` — `python3 scripts/generate.py decisions-md > discovery/state/decisions.md`
2. `discovery/state/governance-decisions.md` — `python3 scripts/generate.py governance-md > discovery/state/governance-decisions.md`
3. `discovery/state/open-questions.md` — `python3 scripts/generate.py questions-md > discovery/state/open-questions.md`
