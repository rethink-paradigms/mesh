Task 8: Implement real reconciliation (Docker + edge cases)
============================================================

Files Modified:
- internal/daemon/daemon.go
- internal/daemon/daemon_test.go
- internal/body/manager.go
- internal/store/store.go
- internal/mcp/mcp_test.go (fixed pre-existing build error)

Implementation Summary:
1. Real reconcile() function in daemon.go:
   - Iterates over all bodies in store
   - Looks up substrate adapter for each body
   - Checks container existence via GetStatus
   - Running/Starting/Stopping + missing container → Error
   - Error + container exists → verify and transition to Running if actually running
   - Migrating + no active migration record → Error
   - Skips bodies without instance_id (never provisioned)
   - Skips substrates not yet wired (e.g. Nomad)

2. Added reconcileSteps counter to Daemon struct:
   - Incremented on each state transition performed by reconcile
   - Exposed in health endpoint response as "reconcile_steps"

3. Added TransitionBody method to BodyManager:
   - Loads body from store, applies transition, persists to store
   - Used by reconcile to perform state transitions

4. Added QueryRow method to Store:
   - Enables direct SQL queries for migration record lookup
   - Used by hasActiveMigration helper

Tests Added (all PASS):
- TestReconcileEmpty: reconcile on empty store (no-op)
- TestReconcileMissingContainer: Running body with missing container → Error
- TestReconcileOrphanedStoreRecord: Error body with missing container → unchanged
- TestReconcileStateMismatch: body without instance_id → skipped
- TestReconcileMigrationRecovery: Migrating body without migration record → Error
- TestReconcileHealthSteps: verify reconcile_steps in health response
