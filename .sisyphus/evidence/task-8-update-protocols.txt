# Update Protocols

> What to do after making changes to the project.
> These protocols are invoked by the mesh-nav skill, not manually.

---

## After Making a Decision

When a new decision is made during a session:

1. **Check for conflicts** first:
   ```bash
   python3 ~/.agents/skills/mesh-nav/scripts/gov.py conflicts <NEW_DECISION_ID>
   ```
   If conflicts found, resolve before proceeding.

2. **Add entity to DB**:
   ```bash
   python3 ~/.agents/skills/mesh-nav/scripts/gov.py add_entity <ID> decision "<TITLE>" \
     --status accepted \
     --body "<full context/decision/rationale>"
   ```

3. **Add edges** for relationships:
   ```bash
   python3 ~/.agents/skills/mesh-nav/scripts/gov.py add_edge <ID> <RELATED_ID> enables
   python3 ~/.agents/skills/mesh-nav/scripts/gov.py add_edge <ID> <RELATED_ID> conflicts_with
   python3 ~/.agents/skills/mesh-nav/scripts/gov.py add_edge <ID> <RELATED_ID> blocks
   ```

4. **Regenerate markdown**:
   ```bash
   python3 ~/.agents/skills/mesh-nav/scripts/generate.py decisions-md > discovery/state/decisions.md
   python3 ~/.agents/skills/mesh-nav/scripts/generate.py governance-md > discovery/state/governance-decisions.md
   ```

5. **Update CONTEXT.md** decision summary section using:
   ```bash
   python3 ~/.agents/skills/mesh-nav/scripts/generate.py context-summary
   ```

---

## After Creating a Research File

1. Create the file in `discovery/research/<topic-slug>.md`
2. Update `discovery/INDEX.md` to reference the new file
3. If the research answers an open question, update the question entity status:
   ```bash
   python3 ~/.agents/skills/mesh-nav/scripts/gov.py update <QUESTION_ID> --status resolved
   python3 ~/.agents/skills/mesh-nav/scripts/generate.py questions-md > discovery/state/open-questions.md
   ```

---

## Session End Protocol

When a session is wrapping up:

1. **Log the session**:
   ```bash
   python3 ~/.agents/skills/mesh-nav/scripts/session.py end --id <SESSION_ID> \
     --summary "<what was done>" \
     --focus "<what was the focus>" \
     --next-steps "<what comes next>" \
     --decisions '["D11"]' \
     --files-created '["path/to/file"]' \
     --files-modified '["path/to/file"]'
   ```

2. **Update SESSION-LOG.md** — add entry at the top (newest first)

3. **If decisions changed**, regenerate markdown:
   ```bash
   python3 ~/.agents/skills/mesh-nav/scripts/generate.py decisions-md > discovery/state/decisions.md
   ```

4. **Update CONTEXT.md** with current state

---

## Drift Detection

On mesh-nav invocation, check for drift between generated markdown and DB:

1. **Generate temp copies** and compare:
   ```bash
   python3 ~/.agents/skills/mesh-nav/scripts/generate.py decisions-md > /tmp/decisions-check.md
   diff discovery/state/decisions.md /tmp/decisions-check.md
   ```
   If different: flag as drifted, offer to regenerate.

2. **Check timestamps**: Compare entity `updated_at` with file modification time.

3. **Flag conditions**:
   - Entity exists in DB but not in generated file → regenerate
   - File exists but entity missing from DB → warn (data loss risk)
   - Content differs → offer to regenerate from DB (DB is source of truth)

---

## After Folder Structure Changes

1. Update `~/.agents/skills/mesh-nav/references/folder-structure.md`
2. Update `discovery/INDEX.md`
3. Run cross-reference check:
   ```bash
   grep -rn "discovery/decisions.md\|discovery/open-questions.md" --include="*.md"
   ```
   Zero stale paths required.

---

## Summary

| Trigger | Protocol |
|---------|----------|
| New decision | gov.py add_entity → add_edge → generate.py → update CONTEXT.md |
| New research | Create file → update INDEX.md → maybe resolve question |
| Session end | session.py end → update SESSION-LOG.md → maybe regenerate |
| mesh-nav invoke | Drift check → flag if DB and markdown diverge |
| Folder change | Update folder-structure.md → update INDEX.md → cross-ref check |
