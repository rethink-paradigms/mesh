## Design Decisions
- Job ID generated from sanitized image name + timestamp (mesh-<image>-<unixtime>)
- Start/Stop use Nomad job scale (count=1 / count=0)
- GetStatus maps Nomad client statuses: pending→Starting, running→Running, failed/lost→Error, complete/terminal→Stopped
- Config reads from env vars NOMAD_ADDR, NOMAD_TOKEN, NOMAD_REGION, NOMAD_NAMESPACE with fallback to defaults
- Exec/Export/Import use Nomad alloc fs/exec APIs with graceful fallback
